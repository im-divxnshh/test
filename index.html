📐 WebSocket + WebRTC Architecture

I designed a lightweight hybrid system:

1. WebSocket Gateway

  • Scaled using Node.js + ws library
  • Handles authentication, room management, and broadcasts
  • Maintains connection state for presence updates

2. Signaling Layer

  • Handles ICE negotiations, offer/answer exchange
  • Fallbacks over WebSocket if STUN/TURN fail

3. Client Integration

  • Hooks into React UI for stateful call controls
  • Typed events for mic/camera toggles, screen share
“A solid architecture hides its complexity through smart design.”

🔄 Connection & Signaling Flow
  1. User joins call → WebSocket handshake created
  2. Client A sends WebRTC offer → server routes to B
  3. Client B replies with answer → A initiates ICE gathers
  4. ICE candidates exchanged real-time via WS
  5. Peer-to-peer media streams established

Graceful fallback detects blocked ports and auto-switches to TURN servers when needed.

“Every negotiation path must account for the worst-case connection failure.”
